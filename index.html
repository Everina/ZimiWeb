01-02 · 2025.04 — 2025.12

嗅嗅豹

面向跨境时尚非标品选品场景的 AI 分析平台。两期都围绕同一类选品问题展开：一期服务内部 Lazada 和 1688 行业运营，验证多 Agent 能否完成商机分析；二期面向外部跨境商家，把高频分析需求沉淀为稳定工作流。

核心洞察
  • 商机定义取决于商家类型，产品要先判断谁在选品，再定义什么算机会。
  • 数据处理链路和搜索策略强耦合，类目映射、CPV 标准化和实体清洗会直接影响召回与排序质量。
  • 高频需求应该工作流化，Agent 处理长尾，真实业务更需要稳定、可评测、低成本的分析路径。
  • Agent 产品需要数据飞轮，把用户反馈和经营结果回收为需求理解、召回、排序和分析能力的优化信号。
一期抽象

内部真实选品需求，本质是多平台数据对齐和商品特征工程。

运营的真实工作流：从 Shein、Temu、Shopee most popular 和 new arrival 等榜单收集候选商品，再人工判断哪些商品特征卖得好、增速快。

Planning Agent
系统大脑：策略制定与整体编排

负责理解运营输入，把模糊选品需求拆成类目、平台、时间、趋势方向等结构化维度，并输出可执行的选品分析方案，调度后续 Search 和 Opportunity Agent。

Search Agent
双路径数据获取：实时爬取 + 结构化召回

实时路径通过 Browser Use 获取当天商品动态数据；结构化查询路径结合 NL2SQL 和 Embedding 双路召回商品数据，再做在线融合、排序和类目映射。

Opportunity Agent
提取商品特征，聚合可解释机会

从召回商品中提取款式、材质、颜色、价格带、场景等关键特征，识别卖得好、增速快的共性模式，为后续排序判断和运营决策提供依据。

一期 Insight / 架构转向

一期跑通了商机分析链路，但也证明开放式 Agent 未必最适合跨境商家

对跨境商家来说，他们需要的往往不是一个自由规划的 AI 助手，而是一条能快速进入、稳定复用、过程可信的选品决策路径。跨境商家本身有很强的工具使用思维，更习惯点击入口后直接看到可判断结果的产品逻辑。因此我们先把他们常见的选品动作抽象为四种方式。

爆品跟卖

用历史交易和销量数据识别已验证需求，在供应链优势前提下判断跟卖成功概率。

新品捕捉

用销量增长率和上新动态捕捉市场热度，优先发现快速增长的潜力商品。

社媒流量

把互动量、内容热度和 KOL 信号转化为市场关注度，识别真实消费兴趣。

品牌追随

分析大牌发布、代言人和市场跟随的时间差，用品牌信号预判未来趋势。

Need 01
用户需要低成本、短链路地完成高频选品

当前 gap：在线获取当天商品数据平均耗时 42 分钟，多 Agent 动态规划和临时数据拉取让等待成本过高。

二期取舍：把确定性强的分析路径前置固化，优先优化离线数据处理、搜索召回和稳定工作流。

Need 02
用户需要把模糊问题快速转成可分析维度

当前 gap：自由文本输入看似灵活，但商家每次都要重新描述品类、地区、趋势方向和筛选条件。

二期取舍：先把爆品跟卖、新品捕捉、社媒流量和品牌追随抽象为选品方式，再收敛为电商、社媒、品牌三类场景和约 12 类固定工作流。

Need 03
用户需要知道为什么值得分析、为什么值得做

当前 gap：报告如果只给结论，不展示数据来源、特征依据和排序逻辑，商家很难用于采购和上架决策。

二期取舍：将理解、召回、类目映射、排序、报告拆成可追踪节点，让分析过程承担“解释机会”的角色。

二期落地

明确三类趋势分析，把主要时间投入多源异构数据处理和搜索优化

二期不再强调“Agent 自由规划”，而是把商家高频选品需求明确为电商、社媒、品牌三类趋势分析。用户看到的是更清晰的报告入口；我们真正花时间攻克的，是背后的多源异构数据处理链路：从数据接入、清洗、类目映射和 CPV 标准化，到一键生成分析报告，并持续反哺搜索召回、排序和评测优化。

数据接入 多源数据接入与清洗

接入平台榜单、社媒趋势、达人内容、品牌数据和搜索热度，先做字段清洗、去重和实体标准化。
数据权重 从单一平台商品数据，转向社媒 + 品牌 + 电商交叉验证

平台数据用于确认需求和价格竞争，社媒数据用于捕捉用户兴趣和内容热度，品牌数据用于判断风格方向、目标人群和趋势扩散。
类目映射 多级类目映射表 + LLM 生成评估闭环

不同平台类目体系差异很大，类目映射决定了数据能否比较。我设计多级类目映射表，并用 LLM 生成候选映射，再通过评估闭环校验。
CPV 标准化 多元异构数据沉淀为统一 CPV 特征层

将商品、帖子、达人和品牌数据里的颜色、材质、风格、场景等特征统一沉淀，降低后续召回和排序的语义噪声。
搜索优化 搜索召回、排序优化与质量评测耦合

数据质量和搜索策略相互影响，因此我们设置了三种难度的搜索评测集，分别评估类目准确率、相关性准确率和召回质量，并按不同难度做多轮迭代。
内测运营

通过 1688 和 AliExpress 建立商家联系，冷启动商家 BD 与内测运营

二期不只是产品和数据链路重构，我也通过 1688、AliExpress 侧资源建立外部商家联系，负责商家触达、沟通、试用引导和反馈回收。我们累计接触 127 位时尚非标品跨境商家，其中在内测阶段自愿使用，且使用天数和使用次数都明显更高的商家共有 12 家。这意味着从真实 BD 商家到高活内测公司的转化接近 9%，是一个比较强的有效需求信号。

核心用户
核心用户分为小 B 商家老板和大中型公司的成熟运营团队

年轻、熟用 AI 的小 B 商家老板更愿意“先试再说”，对新鲜点子、高潜产品方向和弱数据支撑的早期信号都高度敏感，但需求较发散，较难抽象出稳定范式，优先级靠后。大中型公司的成熟运营团队工具心智更强，更强调稳定性、可验证性与流程化，所有判断都需要数据链条、指标逻辑与可重复方法。

付费意愿
大体量独立站商家明确提出私有社媒数据库需求，并表达付费意愿

兰亭集势、唯今国际两家大体量独立站商家都明确提出，希望我们帮助他们建立私有社媒数据库，用于持续追踪内容趋势、达人素材和商品机会，辅助社媒选品决策。这个反馈说明成熟商家不只需要一次性报告，更愿意为可沉淀、可复用的数据资产付费。

阶段成果
形成 12 家双高活跃内测公司，搜索质量也达到可对外验证水平

12 家公司不是被动注册用户，而是在真实内测中自愿使用、天数和次数双高的商家；同时二期评测集中，类目准确率达到 85%，相关性准确率达到 84%，为后续商业化验证提供了可量化基础。

内测反思

内测之后，我们重新判断选品产品的商业化边界

01
产品心智不够聚焦

电商平台、社媒、品牌三种选品入口并行，削弱了核心价值表达，用户难以快速理解产品究竟解决哪个关键问题。

02
社媒选品链路仍需补齐认知

多数商家“想做”社媒趋势选品，但还没有跑通清晰、可复制的方法，产品不仅要给工具，也要解释从趋势识别到选品决策的完整逻辑。

03
目标用户与付费能力错配

中腰部以上平台商家已有稳定链路，外部工具依赖低；大体量独立站数量有限，小 B 商家付费能力和意愿不足，商业化基础受限。

04
能力不够锋利，缺少数据飞轮

商家基于“商机”上架后的销量和运营数据无法回流，既难验证选品效果，也难持续优化模型，产品没有形成正向数据飞轮。

NEXT OPPORTUNITY
更高频、更接近付费的需求，是商家的私有视频库和商品营销视频生成

内测同时让我们看到，商家对私有视频数据库有很强需求：他们希望在自己经营类目下快速找到同款爆款视频，拆解可复用的内容结构，并生成适配自己商品的营销视频。相比选品分析，这个需求发生频次更高，离投放和成交更近，商业价值也更明确。

商品视频生成

从低频选品分析，延展到更高频的商品营销内容生成

在嗅嗅豹内测过程中，我们发现选品分析虽然能帮助商家判断机会，但它在日常经营里相对低频。更高频的需求是：商家已经有一个正在售卖的商品，希望在私有视频库里快速找到经营类目下的同款爆款视频，理解它为什么能爆，并复刻成自己的商品营销视频。

这个方向更接近商业化闭环：内容生成不是终点，视频发布后的曝光、点击、互动、转化和投放成本都可以回流，反过来优化爆款视频召回、脚本拆解、素材选择和生成策略，形成“内容生产 → 投放验证 → 数据回流 → 模型与策略迭代”的数据飞轮。

从内容复刻到投放验证

先在私有视频库中召回同款爆款视频，拆解开头钩子、卖点表达、镜头结构和 CTA，再生成适配商家商品的营销视频，用真实投放表现验证内容质量。

从效果数据回流到策略迭代

把曝光、点击、互动、转化、完播率和成本等指标回收为评测信号，持续优化视频召回排序、脚本模板、素材选择和生成参数。

从单点生成到可复用 Skill

当链路具备明确输入、可验证输出和持续优化信号后，商品营销视频生成就能被封装成 Agent 可调用、可组合、可分发的业务 Skill。

04 · 2026.02 — 至今

虾小宝

2 月随着 OpenClaw 热度，我们快速切入 Agent SkillHub。虾小宝的目标不是让更多人知道一个新产品，而是帮助普通用户意识到日常任务可以交给 Agent，并为 Agent 配齐完成任务所需的能力、身份、协作和交易装备。

核心洞察
  • 增长难点不是曝光，而是用户习惯，普通用户很难主动意识到生活和工作里的任务可以交给 Agent 自动执行。
  • Agent 要替人完成任务，必须先被装备起来，能力、身份、运行环境和协作网络缺一不可。
  • SkillHub 解决“会不会做”，身份资产解决“能不能做”，前者匹配能力，后者打通真实系统。
  • 社区和商城解决“能不能持续变好”，通过 Agent 评论、声誉和开发者供给形成 Marketplace 的信任基础。
产品设计链路

从“Agent 替人完成任务”反推四类装备

如果目标是让 Agent 真正替人类用户完成任务，产品就不能只提供 Skill 列表，而要沿着任务执行链路补齐能力发现、真实身份、可信协作和持续供给。

能力装备
SkillHub

人类任务复杂多样，Agent 首先要知道该调用什么能力。SkillHub 从 intent → skill 推荐、结构化展示、准入评测和沙盒试跑，升级为可推荐、可验证、可运行的能力矩阵。

执行装备
身份资产

找到 Skill 不代表任务能完成，Agent 还会卡在注册、验证、登录、API 调用和额度支付上。因此我们设计独立 Agent 身份，支持邮箱 / 手机号验证、常见 AK 预配置、API Key 托管和 API 钱包。

协作装备
Agent 社区

单个 Agent 的能力有上限，复杂任务需要找到可信 Agent 帮忙。自治社区让优质 Agent 持续使用、评论、推荐能力并参与协作。

供给装备
商城与声誉

平台要持续有好 Skill，就要让开发者能托管和商业化能力，也让 Agent 的真实评论、成功率和稳定性成为推荐排序和交易信任基础。

Product direction

我希望做能够长期演进的 AI 产品

我对 AI 产品的长期兴趣，不只是做一个通用对话入口，而是帮助 AI 能力真正进入用户的日常工作流。很多用户还没有意识到 AI 能帮自己完成什么任务，认知仍停留在问答、搜索和简单内容生成上；但 AI 更大的价值，可能在于任务拆解、流程自动化、决策辅助和跨工具协作。

同时我也认识到，能够主动创作 AI 工作流的用户比例相对较少。因此产品需要找到一个平衡点：一方面通过场景化用户教育激发主动探索，另一方面把少数创作者沉淀出的高质量工作流，通过模板、推荐、案例和低门槛配置释放给更多用户。我希望做的产品，不只是提供模型能力，而是让能力被理解、被复用、被持续迭代。

01 Product expression
把 AI 能力翻译成用户任务

产品不直接展示模型能力，而是用场景、任务和案例告诉用户：这件事可以交给 AI 做。

02 User perspective
让普通用户先得到结果

用户不需要一开始就理解工作流怎么搭建，而是能从可用模板开始，先完成一次真实任务。
