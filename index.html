嗅嗅豹

面向跨境时尚非标品选品场景的 AI 分析平台。两期都围绕同一类选品问题展开：一期服务内部 Lazada 和 1688 行业运营，验证多 Agent 能否完成商机分析；二期面向外部跨境商家，则回到真实选品工作流程，满足商家对效率、稳定性和可信过程的需求，并将高频任务沉淀为工作流，由 Agent 承接长尾和异常问题。

核心洞察
  • 选品不是找商品，而是形成可执行的经营判断，商家真正需要的是从多源信号到采购、上新、跟卖决策的稳定路径。
  • Agent 不该自由规划一切，高频判断要工作流化，长尾、异常和信息补全才交给 Agent 兜底。
  • AI 选品的壁垒不是报告生成，而是可信链路，类目映射、同款识别、CPV 标准化、搜索召回和评测体系共同决定用户是否敢用。
一期抽象

内部真实选品需求，本质是多平台数据对齐和商品特征工程。

运营的真实工作流：从 Shein、Temu、Shopee most popular 和 new arrival 等榜单收集候选商品，再人工判断哪些商品特征卖得好、增速快。

Planning Agent
系统大脑：策略制定与整体编排

负责理解运营输入，把模糊选品需求拆成类目、平台、时间、趋势方向等结构化维度，并输出可执行的选品分析方案，调度后续 Search 和 Opportunity Agent。

Search Agent
双路径数据获取：实时爬取 + 结构化召回

实时路径通过 Browser Use 获取当天商品动态数据；结构化查询路径结合 NL2SQL 和 Embedding 双路召回商品数据，再做在线融合、排序和类目映射。

Opportunity Agent
提取商品特征，聚合可解释机会

从召回商品中提取款式、材质、颜色、价格带、场景等关键特征，识别卖得好、增速快的共性模式，为后续排序判断和运营决策提供依据。

一期 Insight / 架构转向

从内部运营走向外部商家后，我们重新定义了跨境选品的真实场景

外部跨境商家的商机分析比一期内部运营场景更复杂：他们不只看平台商品数据，还会同时参考社媒内容、品牌动向、广告投放和搜索热度。但这类用户也有很强的工具使用思维，更习惯从明确入口进入任务，并快速看到可判断、可复用的结果。因此二期我们先重新拆解商家的选品场景，再定义不同场景下对应的分析需求。

爆品跟卖

用历史交易和销量数据识别已验证需求，在供应链优势前提下判断跟卖成功概率。

新品捕捉

用销量增长率和上新动态捕捉市场热度，优先发现快速增长的潜力商品。

社媒流量

把互动量、内容热度和 KOL 信号转化为市场关注度，识别真实消费兴趣。

品牌追随

分析大牌发布、代言人和市场跟随的时间差，用品牌信号预判未来趋势。

Gap 01
时间成本要更低，分析效果要更稳定

当前 gap：在线获取当天商品数据平均耗时 42 分钟，多 Agent 动态规划和临时数据拉取让用户等待过久，且任一环节波动都会影响最终结果。

Gap 02
自由输入要转成标准化表单

当前 gap：自由文本输入看似灵活，但商家每次都要重新描述品类、地区、趋势方向和筛选条件，系统也难以稳定理解和复用。

Gap 03
用户需要明确“为什么做”

当前 gap：报告如果只给结论，不展示数据准确性、特征依据和排序逻辑，商家很难判断这个机会是否值得采购和上架。

Gap 04
商家需要更多数据源，也需要可对齐、可比较

当前 gap：真实选品判断不只看电商平台，还会参考社媒内容、品牌信息、广告投放和搜索热度；这些数据源各自有不同类目、实体和趋势信号，如果不能识别差异，报告结论就会变成不可比较的拼接。

二期落地

抽象 4 种商机发现思路、3 类数据源和 12 条高频工作流

二期整体约 4.2 个人力，重点投入在场景抽象、报告入口产品化、多源数据处理、搜索评测和商家内测反馈闭环。架构上，我们从一期的通用多 Agent 动态规划，改成 Workflow 承接确定性高频任务、Agent 兜底异常路径和长尾问题。调整后，报告生成时间从一期约 42 分钟缩短到约 2 分钟，链路成功率也从 72% 提升到 97% 以上。

二期的落地重点，是把真实选品动作产品化：用户看到的是更清晰、更快进入的报告入口；我们真正花时间攻克的，是背后的多源异构数据处理链路，让不同来源的数据能够被对齐、比较、生成报告，并持续反哺搜索召回、排序和评测优化。

数据接入 多源数据接入、清洗与交叉验证

接入平台榜单、社媒趋势、达人内容、品牌数据和搜索热度，先做字段清洗、去重和实体标准化；再根据数据源差异分配判断角色：平台数据确认需求和价格竞争，社媒数据捕捉用户兴趣和内容热度，品牌数据判断风格方向、目标人群和趋势扩散。
类目映射 多级类目映射表 + LLM 生成评估闭环

不同平台类目体系差异很大，类目映射决定了数据能否比较。我设计多级类目映射表，并用 LLM 生成候选映射，再通过评估闭环校验。
CPV 标准化 多元异构数据沉淀为统一 CPV 特征层

将商品、帖子、达人和品牌数据里的颜色、材质、风格、场景等特征统一沉淀，降低后续召回和排序的语义噪声。
搜索优化 多阶段召回、Re-ranker 与质量评测闭环

内测阶段我们并行上线三类搜索链路：query 扩展后的联网搜索、基于视觉相似度的图搜，以及候选集召回后的 re-ranker 重排；同时评测标题、OCR、视觉标签、商品属性和脚本结构等信息如何做 embedding，用于视频理解、同款召回和相关性排序。搜索评测集按难度拆分，持续评估类目准确率、相关性准确率和召回质量。
内测运营

通过 1688 和 AliExpress 建立商家联系，冷启动商家 BD 与内测运营

二期不只是产品和数据链路重构，我也通过 1688、AliExpress 侧资源建立外部商家联系，负责公司触达、沟通、试用引导和反馈回收。我们累计接触 127 家时尚非标品跨境商家公司，其中在内测阶段自愿使用，且使用天数和使用次数都明显更高的内测公司共有 12 家。这意味着从真实 BD 公司到高活内测公司的转化接近 9%，是一个比较强的有效需求信号。

核心用户
核心用户分为小 B 商家老板和大中型公司的成熟运营团队

年轻、熟用 AI 的小 B 商家老板更愿意“先试再说”，对新鲜点子、高潜产品方向和弱数据支撑的早期信号都高度敏感，但需求较发散，较难抽象出稳定范式，优先级靠后。大中型公司的成熟运营团队工具心智更强，更强调稳定性、可验证性与流程化，所有判断都需要数据链条、指标逻辑与可重复方法。

付费意愿
大体量独立站商家明确提出私有社媒数据库需求，并表达付费意愿

兰亭集势、唯今国际两家大体量独立站商家都明确提出，希望我们帮助他们建立私有社媒数据库，用于持续追踪内容趋势、达人素材和商品机会，辅助社媒选品决策。这个反馈说明成熟商家不只需要一次性报告，更愿意为可沉淀、可复用的数据资产付费。

阶段成果
形成 12 家双高活跃内测公司，效率、稳定性和搜索质量达到可对外验证水平

12 家内测公司不是被动注册用户，而是在真实内测中自愿使用、天数和次数双高的商家团队；技术方案调整后，报告生成时间从一期约 42 分钟缩短至约 2 分钟，成功率基本稳定在 97% 以上。同时二期评测集中，类目准确率达到 85%，相关性准确率达到 84%；从早期用户反馈“搜索完全不准”，到后期问卷中不再收到明确的搜索问题反馈，也说明搜索满意度和整体可用性有明显提升。

内测反思

内测之后，我们重新判断选品产品的商业化边界

01
产品心智不够聚焦

电商平台、社媒、品牌三种选品入口并行，削弱了核心价值表达，用户难以快速理解产品究竟解决哪个关键问题。

02
社媒选品链路仍需补齐认知

多数商家“想做”社媒趋势选品，但还没有跑通清晰、可复制的方法，产品不仅要给工具，也要解释从趋势识别到选品决策的完整逻辑。

03
目标用户与付费能力错配

中腰部以上平台商家已有稳定链路，外部工具依赖低；大体量独立站数量有限，小 B 商家付费能力和意愿不足，商业化基础受限。

04
能力不够锋利，缺少数据飞轮

商家基于“商机”上架后的销量和运营数据无法回流，既难验证选品效果，也难持续优化模型，产品没有形成正向数据飞轮。

NEXT OPPORTUNITY
更高频、更接近付费的需求，是商家的私有视频库和商品营销视频生成

内测同时让我们看到，商家对私有视频数据库有很强需求：他们希望在自己经营类目下快速找到同款爆款视频，拆解可复用的内容结构，并生成适配自己商品的营销视频。相比选品分析，这个需求发生频次更高，离投放和成交更近，商业价值也更明确。

商品视频生成

从低频选品分析，延展到更高频的商品营销内容生成

在嗅嗅豹内测过程中，我们发现选品分析虽然能帮助商家判断机会，但它在日常经营里相对低频。更高频的需求是：商家已经有一个正在售卖的商品，希望在私有视频库里快速找到经营类目下的同款爆款视频，理解它为什么能爆，并复刻成自己的商品营销视频。

这个方向更接近商业化闭环：内容生成不是终点，视频发布后的曝光、点击、互动、转化和投放成本都可以回流，反过来优化爆款视频召回、脚本拆解、素材选择和生成策略，形成“内容生产 → 投放验证 → 数据回流 → 模型与策略迭代”的数据飞轮。

从内容复刻到投放验证

先在私有视频库中召回同款爆款视频，拆解开头钩子、卖点表达、镜头结构和 CTA，再生成适配商家商品的营销视频，用真实投放表现验证内容质量。

从效果数据回流到策略迭代

把曝光、点击、互动、转化、完播率和成本等指标回收为评测信号，持续优化视频召回排序、脚本模板、素材选择和生成参数。

从单点生成到可复用 Skill

当链路具备明确输入、可验证输出和持续优化信号后，商品营销视频生成就能被封装成 Agent 可调用、可组合、可分发的业务 Skill。

04 · 2026.02 — 至今

虾小宝

虾小宝的产品使命，是帮助 Agent 从“会对话”走向“能执行真实任务”：让 Agent 找到合适能力、拥有可用身份、接入真实工具，并在能力不足时完成协作。它源于一次能力链路的抽象与外溢：我们先将商品营销视频生成封装为可复用 Skill，又发现中文开发者生态缺少 Skill 的标准化分发、协作与交易网络。因此虾小宝从 SkillHub 出发，进一步为 Agent 补齐能力配置、身份绑定、协作和交易装备。

核心洞察
  • Agent 要完成真实任务，必须先被装备起来，能力、身份、权限、额度、运行环境和协作网络缺一不可。
  • SkillHub 不是工具目录，而是 Agent 的能力调用层，重点不是展示 Skill，而是让 Skill 可发现、可验证、可组合、可交易。
  • 增长不是曝光问题，而是任务委托习惯问题，产品要帮助用户意识到：很多工作不是问 AI，而是交给 Agent 执行。
产品设计链路

从“Agent 替人完成任务”反推四类装备

如果目标是让 Agent 真正替人类用户完成任务，产品就不能只提供 Skill 列表，而要沿着任务执行链路补齐能力发现、真实身份、可信协作和持续供给。

能力装备
SkillHub

人类任务复杂多样，Agent 首先要知道该调用什么能力。SkillHub 从 intent → skill 推荐、结构化展示、准入评测和沙盒试跑，升级为可推荐、可验证、可运行的能力矩阵。

执行装备
身份资产

找到 Skill 不代表任务能完成，Agent 还会卡在注册、验证、登录、API 调用和额度支付上。因此我们设计独立 Agent 身份，支持邮箱 / 手机号验证、常见 AK 预配置、API Key 托管和 API 钱包。

协作装备
Agent 社区

单个 Agent 的能力有上限，复杂任务需要找到可信 Agent 帮忙。自治社区让优质 Agent 持续使用、评论、推荐能力并参与协作。

供给装备
商城与声誉

平台要持续有好 Skill，就要让开发者能托管和商业化能力，也让 Agent 的真实评论、成功率和稳定性成为推荐排序和交易信任基础。

PROMOTION CASE

自我评价与未来规划

过去一段时间，我最大的成长，是从“需求执行”转向“问题定义”。在嗅嗅豹、虾小宝这些创新型项目里，我越来越意识到，AI 产品不是越开放越好，而是要先判断哪些地方需要确定性，哪些地方才需要智能弹性。下一阶段，我希望从 Junior PM 继续成长为更成熟的 Senior PM：不只是把需求交付好，而是更早定义问题、判断优先级、推动结果闭环，并长期陪伴一个产品持续演进。

Ideas in motion
Growth 01
从需求执行到问题定义

AI 产品不是越开放越好。更重要的是判断哪些环节需要确定性，哪些地方才需要智能弹性，再把能力组织成用户可信的决策路径。

Growth 02
从上线交付到结果闭环

上线不是结束，而是验证的开始。真正重要的不是功能有没有发布，而是数据、反馈和商业信号是否证明需求值得继续投入。

Growth 03
从单点负责到系统 Owner

创新项目里很多边界并不清晰，所以产品不能只停在需求定义，而要理解原型、链路、效果、用户和增长如何共同影响结果。

Growth 04
把不确定问题变成可验证路径

AI 产品早期常常需求不清、技术边界不稳、评测缺失。关键是先定义判断标准，再把模糊问题拆成能实验、能复盘的步骤。

Product direction

我希望做能够长期演进的 AI 产品

未来我最想做的，是能够长期迭代的 AI 产品。我不希望 AI 产品只停留在通用对话入口，而是希望帮助 AI 能力真正进入用户的日常工作流。很多用户还没有意识到 AI 能帮自己完成什么任务，所以产品需要把 AI 能力翻译成用户任务。

我希望普通用户先通过模板和案例得到结果，再通过推荐、评价、复用和反馈机制，让好的工作流被更多人持续使用。如果用一句话总结我对自己的期待：我希望自己不只是做一个能交付功能的产品经理，而是成为一个能定义真实问题、主动补位、推动长期结果的 AI 产品 Owner。

01 Product expression
把 AI 能力翻译成用户任务

产品不只是展示模型能力，而是用明确入口、模板和案例告诉用户：这件事可以交给 AI 做。

02 Product mechanism
确定性工作流 + 弹性 Agent

把确定性强的高频任务工作流化，把长尾和异常问题交给 Agent 兜底，让产品既稳定又有弹性。

03 Team practice
把个人判断沉淀成机制

继续把需求澄清、原型评审和用户反馈沉淀为稳定机制，让跨团队协作更可复用。
